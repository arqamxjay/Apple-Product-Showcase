# Contributing to Apple Product Showcase

Thank you for your interest in contributing! Here are some guidelines to help you get started.

## How to Contribute

### Reporting Bugs
- Use the GitHub Issues page
- Describe the bug in detail
- Include steps to reproduce
- Mention your browser and OS

### Suggesting Enhancements
- Use GitHub Discussions or Issues
- Clearly describe the enhancement
- Explain the use case and benefits
- Include mockups if applicable

### Submitting Pull Requests
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/YourFeature`
3. Make your changes
4. Test thoroughly in different browsers
5. Commit with clear messages
6. Push to your fork
7. Submit a Pull Request with a detailed description

## Code Style Guidelines

- Use semantic HTML5
- Follow CSS naming conventions (BEM or similar)
- Write clean, commented JavaScript
- Ensure responsive design is maintained
- Test on mobile, tablet, and desktop

## Testing Checklist

Before submitting a PR:
- [ ] Works in Chrome, Safari, Firefox, Edge
- [ ] Responsive on mobile (< 734px)
- [ ] Responsive on tablet (< 1068px)
- [ ] Responsive on desktop (> 1068px)
- [ ] No console errors
- [ ] All links work
- [ ] Images load properly
- [ ] Animations are smooth

## Pull Request Process

1. Update README.md with any new features
2. Test all changes thoroughly
3. Ensure no breaking changes
4. Keep commits clean and organized
5. Provide clear PR description

## Questions?

Feel free to open an Issue if you have any questions!
